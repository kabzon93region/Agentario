# Схема версий Agentario

Формат: **MAJOR.MINOR.PATCH** (например `0.0.5`).

| Часть | Когда повышать | Сброс |
|-------|----------------|-------|
| **MAJOR** (1-я цифра) | Глобальная эволюция проекта | MINOR и PATCH → 0 |
| **MINOR** (2-я цифра) | Добавление, удаление или скрытие функционала | PATCH → 0 |
| **PATCH** (3-я цифра) | Исправления и правки в рамках текущего MINOR | — |

Примеры:

- `0.0.5` → `0.0.6` — исправление бага LM Studio, без нового функционала.
- `0.0.6` → `0.1.0` — новая функция (например, другой onboarding по умолчанию).
- `0.1.0` → `1.0.0` — только по решению владельца проекта (MAJOR).

Версия расширения задаётся в `apps/vscode/package.json` и попадает в VSIX, экран «О программе» и артефакт `release/agentario-<version>.vsix`.
