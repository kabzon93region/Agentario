# Схема версий Agentario

Формат: **MAJOR.MINOR.PATCH** (например `0.2.0`).

| Часть | Когда повышать | Сброс |
|-------|----------------|-------|
| **MAJOR** (1-я цифра) | Глобальная эволюция проекта | MINOR и PATCH → 0 |
| **MINOR** (2-я цифра) | Добавление, удаление или скрытие функционала | PATCH → 0 |
| **PATCH** (3-я цифра) | Исправления и правки в рамках текущего MINOR | — |

## Обязательное правило

**После каждого изменения кода** (функция, исправление, рефакторинг, правка UI, конфигурации расширения) **обязательно** повышать версию и обновлять журнал — даже для незначительных правок.

Минимум: **PATCH** (`0.2.0` → `0.2.1`). Новый функционал: **MINOR**. **MAJOR** — только по решению владельца проекта.

### Чеклист версии (Agentario)

1. `apps/vscode/package.json` → поле `"version"`
2. `CHANGELOG.md` → запись с датой и кратким описанием
3. `README.md` → строка «Версия расширения» и пример имени VSIX
4. **`release/notes/v<x.y.z>.md`** → текст GitHub Release (шаблон: `release/notes/TEMPLATE.md`)
5. Сборка: `build.cmd` → `release/agentario-<version>.vsix`
6. Публикация: `publish-release.cmd` → commit, tag, GitHub Release + VSIX (см. [config/RELEASE.md](config/RELEASE.md))

Примеры:

- `0.1.0` → `0.1.1` — мелкое исправление индексации, правка текста в UI.
- `0.1.0` → `0.2.0` — экспорт чата в Markdown, заголовки сообщений, stats footer.
- `0.2.0` → `1.0.0` — только по решению владельца проекта (MAJOR).

Версия расширения задаётся в `apps/vscode/package.json` и попадает в VSIX, экран «О программе» и артефакт `release/agentario-<version>.vsix`.
